## How the System Works (Overview)

```
Browser
  │
  │  clicks "Run Analysis"
  ▼
Next.js Frontend (cancer-track.vercel.app)
  │
  │  POST /api/predict  ← Next.js API Route (server-side proxy)
  ▼
FastAPI Python Server  (localhost:8000 or Railway/Render)
  │
  ├── StandardScaler.transform(features)
  ├── LogisticRegression.predict_proba(scaled)
  ├── DecisionTree — walks nodes, traces path
  └── returns JSON { lr_prob, dt_result, ensemble, z_scores }
  │
  ▼
Next.js blends result with clinical modifier (age + symptoms)
  │
  ▼
Results displayed in browser
```

The Python server holds the real trained model. The Next.js server proxies requests to it so the Python URL is never exposed to the browser.

## Step-by-Step Setup

### Step 1 — Create the Python folder

In the **root of your CancerTrack repo**, create the `python/` folder and copy the provided files into it:

```
python/
├── __init__.py
├── requirements.txt
├── train.py
└── server.py
```

### Step 2 — Create a Python virtual environment

A virtual environment keeps your Python packages isolated from the rest of your system.

```bash
# From the repo root
python -m venv venv
```

Activate it:

```bash
# macOS / Linux
source venv/bin/activate

# Windows (Command Prompt)
venv\Scripts\activate.bat

# Windows (PowerShell)
venv\Scripts\Activate.ps1
```

You should see `(venv)` appear at the start of your terminal prompt. **Keep this terminal open — you will need it running while developing.**

### Step 3 — Install Python dependencies

```bash
pip install -r python/requirements.txt
```

This installs FastAPI, scikit-learn, uvicorn, and everything else the server needs. It takes about 1–2 minutes the first time.

Verify the install:

```bash
python -c "import sklearn; print('scikit-learn', sklearn.__version__)"
python -c "import fastapi; print('fastapi', fastapi.__version__)"
```

### Step 4 — Train the model

```bash
python python/train.py
```

This script:
1. Loads the Wisconsin Breast Cancer Dataset (built into scikit-learn — no download needed)
2. Trains a **Logistic Regression** and a **Decision Tree** classifier
3. Prints accuracy scores and a 5-fold cross-validation result
4. Saves `python/model.pkl` (the trained model)
5. Saves `python/scaler_params.json` (scaler statistics)

**Expected output:**

```
Loading Wisconsin Breast Cancer Dataset...
  Samples : 569
  Benign  : 357 (62.7%)
  Malignant: 212 (37.3%)

Training Logistic Regression …
  Test accuracy : 0.9561
  Test AUC-ROC  : 0.9921
  5-fold CV     : 0.9561 ± 0.0156

Training Decision Tree …
  Test accuracy : 0.9298
  5-fold CV     : 0.9280 ± 0.0243

✅  Saved model bundle → python/model.pkl
✅  Saved scaler params → python/scaler_params.json
```

> If you see different accuracy numbers that is fine — slight variation is normal across environments.

#### Sync scaler params to frontend (optional but recommended)

Open `python/scaler_params.json` and copy the `SCALER_MEAN` and `SCALER_STD` arrays into `lib/constants.ts`. This keeps the JavaScript fallback model in sync with the real trained values.

### Step 5 — Start the Python server

```bash
uvicorn python.server:app --reload --port 8000
```

The `--reload` flag restarts the server automatically when you edit `server.py`.

**Expected output:**

```
✅  Model loaded — LR acc 0.9561  |  DT acc 0.9298
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

Test it is working:

```bash
curl http://localhost:8000/health
```

```json
{
  "status": "ok",
  "model": "wbcd-lr-dt-ensemble",
  "lr_accuracy": 0.9561,
  "dt_accuracy": 0.9298,
  "n_features": 10
}
```

You can also open `http://localhost:8000/docs` in your browser — FastAPI auto-generates interactive API documentation where you can send test requests.

### Step 6 — Configure the Next.js environment

Copy the template and set the Python server URL:

```bash
cp .env.local.template .env.local
```

The default value is already correct for local development:

```
PYTHON_API_URL=http://localhost:8000
```

### Step 7 — Add the Next.js API route

Create the file `app/api/predict/route.ts` using the provided `route.ts` file.

This file proxies requests from the browser through the Next.js server to your Python server. It also handles timeouts and error messages cleanly.

### Step 8 — Update page.tsx

Replace the `runAnalysis` function in `app/page.tsx` with the version in `app/runAnalysis.snippet.ts`.

Also add a `loading` state:

```typescript
// Add this line with your other useState declarations
const [loading, setLoading] = useState(false);
```

Pass it to `MeasurementsStep`:

```tsx
<MeasurementsStep
  measurements={measurements}
  onChange={updateMeasurement}
  onAnalyze={runAnalysis}
  onBack={() => setStep(2)}
  loading={loading}           // ← add this
/>
```

Then update `MeasurementsStep.tsx` to accept and use `loading`:

```tsx
interface MeasurementsStepProps {
  measurements: Measurements;
  onChange: (key: keyof Measurements, value: string) => void;
  onAnalyze: () => void;
  onBack: () => void;
  loading?: boolean;          // ← add this
}

// On the Run Analysis button:
<button
  onClick={onAnalyze}
  disabled={loading}
  className={`... ${loading ? "opacity-60 cursor-not-allowed" : ""}`}
>
  {loading ? "⏳ Analysing…" : "🔬 Run Full Analysis"}
</button>
```

### Step 9 — Update .gitignore

Add these lines to your `.gitignore` so large or sensitive files are not committed:

```
# Python backend
venv/
python/model.pkl
python/scaler_params.json
python/__pycache__/
*.pyc

# Environment variables
.env.local
```

> `model.pkl` can be 5–10 MB. It is better to regenerate it on the server via `train.py` than to commit it.

### Step 10 — Start both servers

You need **two terminal windows** running simultaneously:

**Terminal 1 — Python (FastAPI):**
```bash
source venv/bin/activate     # or venv\Scripts\activate on Windows
uvicorn python.server:app --reload --port 8000
```

**Terminal 2 — Next.js:**
```bash
npm run dev
```

Open `http://localhost:3000` and run an analysis. The response now comes from the real trained sklearn model.

## Deployment (Railway + Vercel)

When you are ready to deploy:

### Deploy the Python server to Railway

1. Go to https://railway.app and create a new project
2. Connect your GitHub repo
3. Set the **Root Directory** to `/` (repo root)
4. Set the **Start Command** to:
   ```
   pip install -r python/requirements.txt && python python/train.py && uvicorn python.server:app --host 0.0.0.0 --port $PORT
   ```
5. Add environment variable: `ALLOWED_ORIGINS=https://cancer-track.vercel.app`
6. Copy the Railway deployment URL (e.g. `https://cancertrack-production.railway.app`)

### Update Vercel environment variable

In your Vercel project settings → Environment Variables, add:

```
PYTHON_API_URL = https://cancertrack-production.railway.app
```

Redeploy and the full stack is live.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `model.pkl not found` | Run `python python/train.py` first |
| `ModuleNotFoundError: No module named 'fastapi'` | Activate the venv: `source venv/bin/activate` |
| `Connection refused on port 8000` | Start the server: `uvicorn python.server:app --reload --port 8000` |
| `503 Could not reach model server` | The FastAPI server is not running — check Terminal 1 |
| `CORS error in browser` | Add your frontend URL to `ALLOWED_ORIGINS` in `.env.local` |
| `uvicorn: command not found` | The venv is not activated, or install failed — run `pip install uvicorn` |
| Analysis returns identical results every time | Normal if blank fields are all falling back to `SCALER_MEAN` defaults |
